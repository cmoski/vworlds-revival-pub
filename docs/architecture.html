Platform Architecture

Virtual Worlds is a COM/ATL-based platform. All objects are COM components with properties and methods that replicate across clients via the server. The scripting system exposes everything through IDispatch for VBScript/JScript access.

Object Model

The core abstraction is the Thing — a COM object with typed properties and methods. Things are organized in an exemplar hierarchy:

  Thing (base)
    +-- Artifact    (static objects: furniture, walls, decorations)
    +-- Avatar      (user-controlled characters: sprites or 3D models)
    +-- Room        (containers: define space, fog, lighting)
    +-- Portal      (doorways between rooms)
    +-- Actor       (bone-animated characters: skeletal .x models)

Properties replicate across clients automatically. When a client sets Room.Fog = True, the change propagates to the server and all connected clients.

Methods are registered on exemplars and dispatched via FireRemoteMethod / InvokeMethodExt. The system module installs 70+ methods on the base exemplars.

Persistence uses binary .vwc checkpoint files via IVWObjectRepository. World.SaveDatabase serializes the entire world state.
Key Design Decisions

x86 only — code casts pointers to long; 64-bit would require extensive changes
App-local d3drm.dll — legacy Direct3D Retained Mode runtime bundled, deployed alongside binaries
COM/ATL architecture — all objects are COM components, methods registered on exemplars, properties replicate across clients
Headless world creation — cscript.exe with VBScript creates worlds via CreateObject("VWSYSTEM.Client.1")
