API Reference
Core Objects: World, Thing, Room, Avatar, Portal, Artifact
Properties: Vector, Sound, Boundary, PickData, AccessoryData, IntersectData
Methods: System module with 70+ methods registered on exemplars
Scripting: All objects accessible via VBScript or JScript through IDispatch

Key APIs:
World.CreateInstance("Thing", "MyThing") — create objects
Thing.InitializeSpriteGraphics "sprite.spr" — set sprite appearance
Thing.InitializeGraphics "model.x" — set 3D model
Thing.InitializeActor "model.x" — set bone-animated model
Thing.PlayAnimation "anim.anim" — play skeletal animation
Room.Fog = True — toggle room fog
User.Say "Hello" — send chat message
Thing.FindWithin("name") — find child objects
World.SaveDatabase — persist world to .vwc file
Programmer's Guide Topics
Client/Server Architecture — COM-based client connects to server, objects replicate via property marshalling
Object Model — Things have properties and methods, organized in exemplar hierarchy (Thing → Artifact/Avatar/Room/Portal)
Property System — typed properties (String, Long, VARIANT, ObjectProperty) with change notification and replication
Method System — methods registered on exemplars, dispatched via FireRemoteMethod/InvokeMethodExt
Security — per-property and per-method security (disabled in local builds)
Persistence — binary .vwc checkpoint files via IVWObjectRepository
