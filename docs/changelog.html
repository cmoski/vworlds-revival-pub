Changelog — Changes Since the Original v1.5 Release (Feb 2000)

This documents all changes made to modernize the original Visual C++ 6.0 / Windows 98 codebase for Visual Studio 2022 / Windows 11.
Phase 0: CMake Build System
  • Replaced 20+ VC6 .dsp/.dsw project files with a single CMakeLists.txt
  • Auto-discovers MFC/ATL toolsets (including spectre-mitigated libs)
  • All MIDL compilation (49 IDL files) integrated into CMake
  • Win32/x86 target enforced (code casts pointers to long)
Phase 1: Core Platform Compiling
  • vwsystem.dll, sengine.exe, serverV2.exe, vwfound.dll all compile on VS2022
  • Removed atlimpl.cpp/atlctl.cpp/atlwin.cpp includes from all stdafx.cpp (ATL 7+ bundles these)
  • CComModule compat via _ATL_ALLOW_DEPRECATED_FUNCTIONS
  • Added typename for dependent return types in templates (vwtempl.h)
  • Added MFC ConstructElements/DestructElements shims
  • CFileException catch by pointer (not value) in marshbuf.cpp
  • Hoisted for(int i...) variables used outside loop scope
  • OLECHAR* / unsigned short* cast fixes
  • InterlockedCompareExchange → InterlockedCompareExchangePointer in fastcomm
  • Removed issperr.h (merged into sspi.h), wpapi.h, rpcndr.lib/rpcns4.lib
  • CComBSTR::Length → Length(), CFile::hFileNull is HANDLE not UINT
  • strstr const correctness, missing return types, fstream.h → <fstream>
  • HtmlHelp → ::HtmlHelpA (4-arg Win32 form)
  • MIDL strictness: [in,out] → [in] for non-double-indirect interface pointers
Phase 2: D3DRM Stubs & Property Types
  • Created compat/ directory with D3DRM math shims and opaque type stubs for non-graphics targets
  • vwprops.dll built with 5 property types: Vector, Sound, PickData, AccessoryData, IntersectData
  • Boundary property deferred (deep coupling to vwmm resource IDs)
Phase 3: OCX Controls
  • objexplr.dll (Object Explorer) compiles and registers
  • uictrl.dll compiles (VWSelfVw initially excluded, later fixed)
  • VWSelfView root cause: compat/ddraw.h stub shadowed real ddraw.h; fixed with include_d3drm_only + DIRECT3D_VERSION=0x0700
Phase 4: Static Libraries
  • jpeglib.lib (JPEG codec) and d2dutils.lib (utility library) compile
Phase 5: Real D3DRM Runtime
  • Bundled legacy d3drm.dll (352KB) from original DirectX runtime in third_party/d3drm_legacy/x86/
  • vwmm.dll built against real D3DRM headers/libs — 21 CLSIDs registered
  • vwstudio.dll (studio authoring tools) compiles
  • vwrendvw.dll (Render View OCX) compiles
  • CoCreateInstance succeeds for VWRenderRoot, Vector, VWGeometry
  • Removed _MERGE_PROXYSTUB from vwmm (not needed for in-process COM)
  • CLSID_Boundary integrated into vwmm.dll
Phase 6: World Creation
  • HashKey<BSTR> fix (propbase.h) — MFC's default template hashes the pointer, not the string content. Changed to proper template specialization. This was the root cause of all "Property doesn't exist" failures
  • First testworld.vwc (39KB) created by sengine.exe
  • System module installed with 70+ methods and properties
  • serverV2.exe loads the world and reports "ready to receive users"
  • Headless world creation: cscript.exe with VBScript replaces sengine — CreateObject("VWSYSTEM.Client.1") works standalone
Phase 7: Rendering, Multi-User, Content
Rendering:
  • renderhost.exe: MFC-based render host via CWnd::CreateControl on VWRenderView OCX
  • D3DRM initializes, DirectDraw surfaces created, paint thread running at ~60fps
  • Client connects to server, enters world, renders scene
  • Sprite textures load: default.spr → default.gif for avatars and artifacts
  • 3D textured objects: .x files with TextureFilename references load via async callback chain
  • Background rendering: windows keep rendering when alt-tabbed
Multi-user:
  • Two renderhost clients (Alice + Bob) connect to same server, see each other's avatar sprites
  • Cross-client chat via Say/Tell events with CClientEventSink (IVWClientSite vtable + IDispatch)
  • OnUserConnect/OnUserDisconnect show join/leave messages
World content:
  • 5 renderable worlds: Gallery, Home, Landscape, Office, DesktopWorld
  • DesktopWorld: 60+ .X models (lobby, walls, ceiling, patio, tables, furniture) + Music Garden
  • All coordinates from original SDK wizard kits
  • Room geometry fix: Rooms must have GeometryName = "" (empty) — room itself is invisible
Connection stability:
  • Property map atomic swap during UnMarshall — eliminated 21 "Invalid property map" errors per connection
  • EnableSecurity silenced — eliminated 21 "Property doesn't exist" errors per connection
  • Content path fix: Client\Shared\ added to InetFileManager search paths
  • CreateTools resilience: tolerates pre-existing tools via get_ToolExt fallback
  • ReportError rate limiting: first occurrence logged, duplicates suppressed with periodic summary
Object Explorer:
  • ThingTree + TPList OCX hosted as split popout window
  • Click objects in tree → properties update in real-time
  • USE_CONTROLMANAGER must be defined on objexplr.dll
Sound:
  • DirectSound initializes, downloads WAVs via InetFile, plays through speakers
  • Ambient room audio with looping support
Studio editing:
  • vwstudio.dll loads via --edit flag
  • Select, Translate3D, Rotate3D, Scale3D, Zoom tools
  • Right-click context menus on objects
Persistence:
  • World.SaveDatabase writes .vwc checkpoint via IVWObjectRepository::Commit()
  • Ctrl+S in renderhost saves; worlds reload on restart
Avatar customization:
  • --avatar alice.spr flag with InitializeSpriteGraphics
  • Full animated sprites: alice, bob, charles, eve, gracie, kevin, etc.
Command Window:
  • Live VBScript console via --cmdwin flag
  • Raw IActiveScript/IActiveScriptParse hosting
  • World object exposed with SCRIPTITEM_GLOBALMEMBERS
  • World.Global.DefaultRoom.Fog = True changes 3D view in real-time
Original HTML client:
  • webclient.exe hosts WebBrowser (IE/Trident) OCX
  • Navigates to Local Content\Client\Basic\Client.htm
  • Login, rendering, Things tab, sound all functional
Bone animation (Actor exemplar):
  • Wired up the NULL pfUpdateJoint callback to D3DRM frame transforms
  • Actor exemplar with InitializeActor, PlayAnimation, SetJointRotation
  • .anim file format: per-frame euler rotations + positions per bone
  • Joint re-init on world reload, rest-pose reset after animation
Security changes:
  • SUPPORT_CHECKPROPERTYSECURITY and SUPPORT_INVOKEONPROPERTYACCESS disabled for local use
  • SaveDatabase owner check disabled
Automation:
  • launch_world.bat [name] — any world
  • launch_multiuser.bat — two-client test
  • launch_editor.bat — editor with studio tools
  • create_allworlds.vbs / create_desktopworld.vbs — headless creation
Phase 8: Polish (TODO)
  • Safe CRT migration (108 unsafe calls: strcpy, sprintf, etc.)
  • VersionHelpers modernization
  • Unicode support
