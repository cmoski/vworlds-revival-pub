Step One: Detecting Chat

Virtual Worlds objects can be programmed to react in response to chat text that users are typing. The Lesson_4 object has a custom property called secret word.
a Look at the property in the Object Explorer to find the word and figure out a sentence that uses it.

a See if the object responds predictably to the secret word, and wait until it stops.

$ Look at the script subroutine below, which is used by the method, OnPeerTell, that was added to the object Lesson_4. OnPeerTell is an overridable Virtual Worlds event handling method that intercepts text from the chat frame. There is script code within the method that parses a single word from a group of words; this allows the object to "hear" its word in any context. Step Three: More on Timers

This method looks similar to the timer method in Lesson 3. It is used to move an object around, however, and it calls a custom method, end_Surprise, which deactivates sound and returns the object to its original position. When making scripts that control an object's movement and behavior, include script methods that reset the object to its initial state, even if it is only used for testing purposes.

It is possible to make this object fly without stopping (at least while a user is logged into the world), by removing the ability of the timer script to check for conditions that will eventually signal the object to stop itself. In such a case, the timer simply runs as long as the world is loaded in the browser. It is unclear where the bird object would end up; it might take several minutes just to get to it!

Step 4 of 4
Step Four: Virtual Worlds Events, Event Handlers, and Sub Arguments

At this point, you have used two vworlds event handling methods, OnLButtonUp and OnPeerTell to get useful input from a Virtual Worlds user to objects in your world. Virtual Worlds uses many different events, sending them to the objects that can use them to direct their behavior. There are literally dozens of events that are sent in different contexts while a world is running (for example, from objects to their containers, from containers to the objects that make up their contents, and even from room to room).
a Take a close look at the Programmer's Guide to learn about many more useful events that your objects can detect! By this point, you should be able to override the pre-defined event-handlings methods that handle events we haven't discussed, and use them in your own worlds.
~ It is very important to understand from the previous lessons that these event handling methods, which we have overridden, have specific data that they need to use, and the data must be received correctly by the method's script subroutine.
~ You MUST include the correct number of variables used by the predefined method included within your subroutine in the parentheses next to the declaration "sub". These variables are known as "arguments" — the most common reason your scripts may fail to function correctly is that they omit the arguments that are required for the method, or do not use them correctly. It is not necessary to use the arguments, although they provide useful data for your objects in almost every case; you must, however, include an argument list in the subroutine that receives them.
a Study the arguments provided for the event handlers in the Programming Guide carefully.
