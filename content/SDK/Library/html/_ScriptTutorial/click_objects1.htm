Step Two: Overriding OnLButtonUp

Below is script from the module 'lesson_2'. This module points at the file lesson_2_clickable_objects.vbs in your sdk/library/vbmodules directory. This subroutine "overrides" the method OnLButtonUp that Virtual Worlds objects use to detect a mouse click. The code also keeps track of the last time a user clicked this object, to turn its custom method behaviors on and off with the custom methods do_Surprise and stop_Surprise. Read the comments in the code, see if you can follow it, and take a look at the other methods this "click" subroutine calls over the next two steps.

'use this to avoid problems with misspelled
'variable names under this option

'variables using the 'dim' statement prior

'can be saved and reused for as long as
'the client is running vworlds.

'This global variable will hold a reference
'to a sound that can be set once and reused

'time. Later we will show how to do this
'by using a custom property on a virtual
'worlds object instead.

'These three arguments must be included
'in order to override a button click
'event from the mouse on any object.
'They refer to the renderer's window
'handle (not used), and the x and y spot
'in the render window where a user clicked.
     World.Trace "click received at coordinates "&cstr(x)&","&cstr(y)&" of the window..."
     if isobject(box_sound) = false then 'script comments
'here the use of 'isobject' is a way to
'detect if we have already initialized
'the variable box_sound with our sound yet.
'if isobject returns false, it means the
'variable does not have a sound in it, so
'we should set the variable with a sound
'before we try to use it.
          set box_sound = world.CreateSound _
("SDK\library\sounds\_ScriptTutorial\minuet.wav", false, true)
     end if
if my_state <> true then 		 
          This.do_Surprise 'script comments
'This line calls a custom vbscript method on
'the box (doSurprise)
          my_state = true

Step 3 of 4
Step Three: Custom Methods

The following subroutines are custom methods attached to the clickable box object, and they are all called by its OnLButtonDown method. They do not "override" existing Virtual Worlds functionality as the previous "click" subroutine does; once these methods are attached to an object, they can be run on that object just as if they were part of the functionality available to all Virtual Worlds objects.

 


sub do_Surprise
 'script comments
'often world.user.container is a good way to
'get hold of the room object,
'and thus to find and use other objects
'that are contained within it.
'If you know the name of the object
'you want to manipulate in script, you
'can use the function "findwithin" on the
'room to get to it from a vbscript and
'change its properties.
Dim Myvector 'script comments
'we use vectors for much of the motion and
'position functionality
set Myvector = world.CreateVector (0,1,0)
dim clickable_box, rot, rev, dolph
set clickable_box = world. User.container.findwithin("clickable_box")
set rot = world.user. Container.FindWithin("rotator")
set rev = world.user.container. FindWithin("revolver")
set dolph = world.user.container.findwithin("dolphin")
clickable_box.Sound = box_sound
clickable_box.RenderQuality = 3
clickable_box. RotationAxis = MyVector
clickable_box. RotationRate = .05
rev.rotationaxis = MyVector
rev.rotationrate = -.07
Myvector.set 0,0,1
 'script comments
'we can re-use this vector for other values by
'calling 'set' on it
rot.rotationaxis = MyVector
rot.rotationrate = csng(.1)
dolph.renderquality = 3
dolph. IsVisible = true
dolph.PositionDuration = 2000
dolph.Position.y = .5

end sub
   
sub stop_Surprise
 'script comments
'basically, we need to restore most of the
'values for the movements back and
'turn off the sound we were playing
dim clickable_box, rot, rev, dolph
set clickable_box = world.user.container.findwithin("clickable_box")
set rot = world.user.container.findwithin("rotator")
set rev = world.user.container.findwithin("revolver")
set dolph = world.user.container.findwithin("dolphin")

clickable_box.sound = nothing
clickable_box.rotationrate = 0.0
rev.rotationrate = 0.0
rot.rotationrate = 0.0
dolph.isvisible = false
dolph.positionduration = 0
dolph.position.y = 0
end sub  

Step 4 of 4
Step Four: Building Complexity

An object's ability to respond to mouse clicks can be added on, independently of other scripted behaviors, because an object can have any number of methods attached to it to do different things. OnLButtonUp is just one interactive capability that takes advantage of a Virtual Worlds feature, and it could be used to trigger any number of different custom methods, even if some of the methods are on other objects. Later lessons will use multiple methods and multiple objects together to create more complex interactions.
