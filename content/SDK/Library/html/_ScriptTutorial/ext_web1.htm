Step 1 of 5
Step One: Web Interfaces

One of the most useful ways that world builders can provide user control for objects in Virtual Worlds is through the use of external HTML pages with custom scripts embedded in them. A virtual world can display any web page that is located at a valid URL, including local files on a user's machine. To make those web pages act as an interface with Virtual Worlds, special script code must be included in them that enables the page and the virtual world to interact with each other.

Any Virtual Worlds object can call up web pages in a floating window via script.
a To see an example of this behavior, click on the ant in the Virtual Worlds window and try out the controls provided on the page. When you are comfortable, move on to Step 2.
~ It is important to understand that a web page with Virtual Worlds scripting in it can only be effective while a world is up and running in the browser. Since it is specifically designed to accompany a virtual world, its scripts will usually fail to function properly when the page is loaded by itself in the main browser window.

Step 2 of 5
Step Two: Loading a Web Page

Below you can see the first of the two short methods in the module lesson_5_web_objects.vbs. The first method, ant_click, overrides the OnLButtonUp method of a virtual world object, as you have seen in previous examples. It is called when you click on the ant, and contains just a very simple method with one script call, to open, position, and size a floating HTML window with a given URL. Move on to the next step to see the second custom method for the ant object.

 


sub ant_click (hwnd, x, y)
world.trace "sub ant-click called on object "&this.name&"..."
world.ShowURLEx "Ant Control", "Ant Control", "_
sdk\library\html\_ScriptTutorial\tutexample.htm", world.user, 80,80, 300,350

dim result
result = this.invokesupermethod(Array(hwnd,x,y))
end sub
Step 3 of 5
Step Three: Talking to a Web Page

The other method in the file, ant_done_moving, overrides the OnPositionDone method, which is called on any object only when it has finished moving from one position to another, or, in other words, any time that you set an object's position.

When this subroutine actually gets executed depends on how long you have determined an object's position changes should take, which can be customized via its PositionDuration property. The longer the object takes to complete a position change, the longer the duration between setting the position and executing this subroutine.
a To verify this, try changing the PositionDuration property of the Lesson_5 object (the ant) with the Object Explorer, and use the web page to move the ant again.
$ The fireUIEvent call inside the ant_done_moving subroutine is the conduit for the object to talk back to the web page that is controlling it.
a You should be able to control the ant's movement from the web page buttons, and when the movement is completed, this fireUIEvent passes some data back to the web page in a special UIEvent event, containing the name of your custom event as a string.

 


sub ant_done_moving ()
world.trace "I'm done moving"
world.fireUIevent this, "ant_control_showposition"

Step 4 of 5
Step Four: Designing Pages for Virtual Worlds

We have seen code that responds when an object is done moving, but where is the code that actually sets the ant's new position to move it? In the next step, you will see that we have moved much of the functionality of the ant object into the scripts of the web page Ant Control.
~ It is important to note from the last step that the data being passed back to the web page is arbitrarily determined by the ant_done_moving script in the last step, and could be anything that might be used by the web page.

Combining these web-to-virtual world and virtual world-to-web communication techniques with dynamic HTML (DHTML) can provide very interesting graphical interfaces, displays and animations in the web page that can also be used in conjunction with a virtual world. In future lessons, we will describe how to carry the actual HTML source for a web page inside the properties of a Virtual Worlds object. Also, we will look at ways to create an object that can write its own HTML page actively from its scripts, even one that has a custom interface with which a user can control it.
