Step 1 of 8
Step One: Property Animations

In the previous examples in the tutorial you saw objects move in fairly rudimentary ways to give feedback users. Most of these animations were achieved by changing the property of an object while indicating a duration with a separate Durationproperty specific to the property we wanted to animate.

Virtual Worlds 1.5 objects can perform complex animations on many of their graphical properties that change the properties over time along a predetermined path. They can also perform animations that are built into their graphics files, where the animation data is imbedded in the .3-Ds or .X file format. Properties that can be animated are documented in the Virtual Worlds API under the method StartAnimation, and include among others the Position, Scale, and Direction of an object.
a Right click the Orange portal and select Go Through to move into the animation room.
a Move to the area near the bridge and click the cube object, labeled Animate Cube. The OnLButtonUp method attached to the cube will trigger a simple animation that moves the cube along a path.
$ Virtual Worlds has the ability to put text over any object dynamically by manipulating its Label property.
a Try to click the cube while it is moving. The animation should restart each time you click it.
a Watch the Command Window as you click the cube and you will see when the animation starts and whether it completes successfully or is interrupted. Try waiting for the cube to complete and see the message traced in the Command Window.
~ This message is traced by another method on the cube that is specified as the Callback method in the call to StartAnimation. This method is invoked automatically when the animation completes.
a If the animation is interrupted by a new animation, this callback is also invoked but it is passed a parameter that indicates the animation was stopped before completing. Animations can also be set to loop indefinitely, in which case the Callback method is only called when the animation is stopped programmatically.

The animation path consists of a series of position vectors stored in a PropertyMap, which is stored in the SimpleAnimMap property of the cube object.

a Open the Object Explorer, select the object named 'cube' and look for its SimpleAnimMap property. Notice it is of type Property Map. Double-click the property to view the contents of the map in the property map dialog box. You will see a sequence of vectors each labeled with a numeric key. These key value pairs determine the sequence and the relative timing of the motion of the cube, as they are applied to its Position property.
$ Notice that the first and last vectors in the PropertyMap, the ones at the lowest and highest key numbers, are the same. This creates the illusion of a cyclical animation that returns to where it started, so that the next time it is started it appears to be moving from its current position.
~ Animations do not take into account the actual position of the object when they start unless programmed to do so. If the first key (the 'zero' key) is not the same as the actual position of the object, it will appear to jump, as is the case when you click the cube before it has returned to the last position in the path and the animation is complete.

The key values in the property map specify keyframes for the animation. The cube will smoothly move through each of the vectors in the map. Note that the keyframe values range from 0 to 8. This timing information is relative (i.e. the cube does not take 8 seconds to complete the cycle of keyframes) and depends on additional timescale information provided in the call to StartAnimation.

In the next step we will examine the VBScript code for the cube that animates it.

Step 2 of 8
Step Two: Property Animations Continued...

The animation is started by the cube object's OnLButtonUp method found in the script module lesson_9_animations.vbs:

sub cube_onLButtonUp (hWnd,x,y) 'script comments
' this method animates the cube object on a
' simple path when clicked. note that timescale
' value passed in StartAnimation is in keyframes
' per millisecond: this animation runs at .001
' keyframes per millisecond, equivalent to 1
' keyframe per second.
     dim time
     time = world.getanimationsystemtime()
 'script comments
' the getanimationsystemtime methods returns
' the time in ticks since the world was created.
     world.trace "cube animation started at time "& time & " ticks."
this.startAnimation "position",this.simpleAnimMap,"animationDone",A_SPLINE,.001,0,8
end sub

$ Pay close attention to the parameters passed to StartAnimation. There are a lot of parameters and together they provide very precise control of the cube's movement. Many are optional and omitting them produces different results. See StartAnimation in the Virtual Worlds API for details.

1. Position The first parameter "position" indicates that the animation will affect the object's position. There are many other graphical properties that can also be animated.
2. PropertyMap Name The second parameter ("this.simpleAnimMap") specifies the PropertyMap containing the position vectors as the path used for this animation.
3. Callback Method Name The third parameter ("animationDone") specifies a callback method, which is a method of the animating object that will execute when the animation completes or is interrupted.
4. Option Flags The fourth parameter (A_SPLINE) is a flag that indicates that the animation process will use spline functions to interpolate smoothly between the points in the PropertyMap.
5. Time Scale The fifth parameter (.001) specifies the speed that the animation will run in keyframes per millisecond, also known as its 'timescale'. The value .001 produces an animation speed of .001 keyframes per millisecond, which is one keyframe per second. Since the highest key value in the PropertyMap is "8", at this timescale the animation will take 8 seconds to run completely if it is allowed to run through all of the keyframes.
6 & 7. Start Range and End Range The sixth and seventh parameters (0) and (8) indicate the start and end ranges of the animation applied to this path. These are read by StartAnimation as the first and last keyframe values to process. In this case we are running the animation from start to finish.
8. Start Index The eighth parameter (0) indicates the starting index in within the specified ranges. In this case we are starting at the beginning of the range.

$ You can call StartAnimation directly on any object from the to test an animation.
a Try executing this animation by typing the following line into the Command Window:
     dim obj : set obj = user.container.findwithin("cube")
     obj.startAnimation "position",obj.simpleAnimMap,"animationDone", _
      A_SPLINE,.001,0,8,0
$ The cube should appear to animate in a fashion similar to when you clicked on it in the 3-D view.
a Try changing the last 4 parameters and watch how the animation behaves differently. For example, changing the speed to .002 will make the animation run twice as fast. Changing the last 3 parameters will start or stop the animation at different points along the path.
$ Watch the property 'position' on the cube object in the Object Explorer. Notice that the object's position is set to the closest key value in the property map indicated by its end range.

Step 3 of 8
Step Three: File Animations

Some graphics files contain built-in animation information which Virtual Worlds can play by specifying "file" as the animation type when calling StartAnimation. A file animation displays animations embedded in the graphics file itself, at a specified timescale.
~ These animations do not affect the graphical properties of the object in Virtual Worlds, although objects can appear to move, turn, and scale in the renderer in ways that could make a user believe these properties have been altered.
a Find and click the object labeled "grow tree" in the 3-D view. The object will change its geometry smoothly, expand and grow into a different shape.
$ All of the parameters to perform this transformation were applied at the time the model for this object was created. They cannot be viewed or edited in Virtual Worlds but must be altered at the design stage of the graphical model itself!
$ If you click the tree again while it is growing, the animation will restart. If you click the tree after it has stopped growing, it will return to its original "seed" state. This behavior is determined by the object's OnLButtonUp method which you can view below:

dim isTreeGrown 'script comments
' this global variable keeps the state
' of the animation of the tree object
' between multiple executions of the
' method below.
sub treeAnim_onLButtonUp (hWnd,x,y)
 		'script comments
' reset the tree by setting the tree's
' animation index to zero. if an animation
' has finished, this index set still works
' only if its a file animation
     if isTreeGrown then
          this.setAnimationIndex "file",0
isTreeGrown = false
world.trace "tree file animation index set to 0 at time " & _
  world.getanimationsystemtime() & " ticks."
     else 'script comments
' grow the tree by running the tree's
' file animation
          this.startAnimation "file",,,A_SPLINE+A_LOCAL,0.01,0,80
isTreeGrown = true
world.trace "tree file animation started at time " & _
  world.getanimationsystemtime() & " ticks."
     end if
end sub

$ Note that in the call to StartAnimation we did not pass a property map as a parameter as we did with position. File animations do not accept a PropertyMap since their animating data is actually imbedded in the graphics file.
$ Also note that in this example we have specified no callback method. In this case we do not wish to run any methods when the object has completed its animation or is interrupted. In the fourth parameter which accepts the option flags for the animation, we are combining A_SPLINE and A_LOCAL. The A_LOCAL flag causes the animation to run on the client-side. Only the client that is running the animation will see the object change. Removing the A_LOCAL flag will cause all other clients logged into the room to see the object animate on their machines as well.

Step Four: Dynamic Lighting

Virtual Worlds supports the use of dynamic lighting of the 3-D space with an exemplar called Light. Instances of the Light Exemplar can be positioned and edited to create a multi-colored lit landscape in the 3-D view.

Take a moment to familiarize yourself with the special properties of the Light exemplar, listed alphabetically and beginning with the word 'Light'. Virtual Worlds supports multiple types of lights and you can customize the color, attenuation, type, and the width of a light if it is a spotlight.
a Locate the object labeled 'animate light color' in the 3-D view. Its name in the Object Explorer is DirectionalLight
~ The label of a Virtual Worlds object can be any string; it does not have to refer to the object's name!
a Click the object labeled animate light color and notice how the color of the lighting changes. The onLButtonUp method of the directionalLight object runs a LightColor animation.
a Switch to the Geometry Editor by selecting that tab from the tabs immediately below the 3-D view, and make sure that the object 'directionalLight' appears as the selected (highlighted) object in the 3-D view. You may select it by choosing the Select tool from the tool menu immediately above the 3-D view, and then clicking on it in the 3-D view.
a With the directionalLight selected, choose the Rotate tool from the tool menu immediately above the 3-D view (you must be in the geometry edit mode by selecting it from the tabs below the 3-D view to accomplish this).
a Hold the mouse down on the light object and change its direction by dragging the mouse around.
$ Notice how the lighting direction changes on all the surfaces in the 3-D view. This is the nature of a light with a LightType property of '3' , which indicates a directional light. Moving the position of a light with this LightType will not affect the lighting, only its orientation. This creates a nice atmospheric light effect on all the objects. Step 5 of 8
Step Five: Animating Lights

Virtual Worlds supports animating the LightColor and LightAttenuation properties of light objects. In this example we will animate the LightColor property of a light. For an in-depth look at attenuation and its effects, see the Virtual Worlds Documentation for the Light exemplar.
$ As with all property animations, The color values for animating the LightColor property are stored as vectors in a PropertyMap, with numeric keys to indicate the sequence and relative timing.
a Double-click the LightAnimMap property of the DirectionalLight object in the Object Explorer to view the 'path' of light colors performed by the animation. Notice that it is numbered in much higher keyframes than the previous property map. This is arbitrary due to the interaction of the map and the timescale value passed to StartAnimation.
$ This animation is very similar to the position animation applied to the cube earlier. All we have done differently is to supply a different map and property name for the animation to use. All complex animations for virtual worlds use the StartAnimation method as their basis.
a Try clicking on the light in the 3-D view to activate its LightColor animation.
a Take a look at the script below for the DirectionalLight object's onLButtonUp method, found in the script module lesson_9_animations.vbs:

sub directionalLight_onLButtonUp (hWnd,x,y) 'script comments
' this method animates the lightColor property
' of the DirectionalLight object notice that the
' timescale for this animation is one keyframe
' per millisecond, because the points in the path
' are spaced very widely, i.e. key values in
' thousands.
     this.startAnimation "lightColor", this.lightAnimMap, _
  , A_SPLINE+A_LOCAL, 1, 0, 9000
world.trace "directionalLight animation started at time " & _
  world.getanimationsystemtime() & " ticks."
end sub

$ Remember that the keyframe values in the property map were numbered in the thousands, from 0 to 9000. The speed value in the StartAnimation statement is 1, which is substantially different than the timescale used in the previous animation examples. This value of 1 means a timescale of one keyframe per millisecond, or 1000 keyframes per second. At this timescale the whole animation will play in about 9 seconds, because it has nine thousand milliseconds of keyframe times specified in the property map.
~ Numbering the map with higher values gives you much higher resolution timing. For example you could add a keyframe at key 7005, which would play just 5 milliseconds after the keyframe at 7000!

Step 6 of 8
Step Six: Advanced Animations

Multiple properties can be animated simultaneously on a single object (though only one of each property at a time), and multiple objects can be animated along the same path. This functionality can be used (among many other uses) to give the appearance of a single complex articulated object, from animating simple component objects with varying animation instructions.

$ Find the moray-eel in the 3-D view whose head is labeled 'head'. It should be comprised of several distinct labeled body parts, each slowly moving in a sequence about the room. The resulting composite 'fish' swimming slowly around in this lesson demonstrates both of these multi-property and multi-object techniques.
a Click the object labeled "show/hide path". Wait a moment as the script running on this object can take a while to complete at first. You should see a series of polygons showing the points of the fish body parts' shared position path. The short side of the triangle indicates the actual point of a vector in the property map used by these body part objects to animate. Clicking on the object labeled show/hide path subsequently draws and hides these points.
a Now find the object labeled "start/stop fish" in the 3-D view and click it. You should notice the fish jump to an initial position almost immediately and start to move around the path. it will continue to animate past the starting position because the call to StartAnimation on each body part included the A_LOOP flag in its options.

Step Seven: Animation Timing
a Click the object labeled start/stop fish again. Since the StartAnimation call specifies the A_LOCAL flag as an option, clicking the “start/stop fish” object stops the position and direction animations on the fish parts immediately. The A_LOCAL flag provides much faster timing since it runs only on this client, and does not synchronize with other clients.

Each of the fish body part objects runs their own position and direction animations with slightly different timing to appear as though they are moving together. All of the fish body parts use the same position PropertyMap and the same direction PropertyMap, which are properties of the Room object.

a Open the Object Explorer and select the Room object lesson 9-Animation room. Locate the properties 'FishDirMap' and 'FishPosMap' in the properties of the room. Double-click each property to show the lists of keyframes (vectors) that will apply to the objects as they animate along the path.
$ You may notice that the timing of these keyframes is much more detailed than in previous examples. this produces very subtle details of acceleration and deceleration as the animation is run at a given timescale, and shows the detail with which a complex animation may be planned out.
Starting the animations is a matter of doing a series of StartAnimation calls on all the objects one immediately after the other to achieve adequate timing. To space the objects apart, their animations are started with different starting index values along the same path. Here is the code from the subroutine StartFish in lesson_9_animations.vbs:
$ Note that simply by changing the index value into the same animating path at identical speeds, the objects can be made to 'follow' each other.

Step 8 of 8
Step Eight: Animating Avatars and 'Chasing' Objects

Avatars can be animated just like other objects. This chasing animation demonstrates animating the user's avatar and the body parts of the fish simultaneously, thus making the user follow the fish around the room.

Click the object labeled "chase fish" and watch as your avatar follows the fish around the room once and ends up near the starting point.

$ Notice that while your avatar is being animated you cannot navigate with your mouse or keyboard. VWorlds is modifying your avatar's position and direction and it overrides any manual changes you make until the path has completed. This technique is very useful for guiding avatars on a cinematic tour of a world or taking them from object to object in a pleasing manner.
~ Use caution when animating avatars, especially if you are not animating them locally using the A_LOCAL flag. Animations stored on the server, especially ones that run indefinitely using the A_LOOP flag, can be difficult to get out of and frustrating for the user whose body is being controlled by the animation.
a To view the paths used for animating the avatar's position and direction, select the the room object lesson 9-Animation room in the Object Explorer. Locate the properties ChaseDirMap and ChasePosMap in the properties of the room. Double-click each property to show the lists of keyframes (vectors) that will apply to the objects as they animate along the path.

The chasing effect is accomplished by starting position and direction animations on the avatar immediately after starting the fish animations. Rather than have the avatar simply follow the same path as the fish, another path was generated that makes the effect of looking at the fish more realistic than following its exact path.

Since the keyframes of the avatar path were less complex than the path of the fish, a straight forward series of numbers can be used for the keyframe values. Running these animations at .001 keyframes per millisecond (i.e. 1 keyframe per second) should play back the movements of an observer of the fish in sync with the fish itself.

The OnLButtonUp method of the chaseSwitch object runs the fish animations and user avatar animations very close together and with A_LOCAL selected as an option for accurate timing:

sub chaseSwitch_onLButtonUp (hWnd,x,y) 'script comments
' this chase_switch object method starts or
' stops animations on the fish body parts
' and also the user's avatar, using different
' maps for a cinematic effect.
     user.container.chasePosMap.property("0") = world.createVector _
  (user.position.x,user.position.y,user.position.z)
user.container.chaseDirMap.property("0") = world.createVector _
  (user.direction.x,user.direction.y,user.direction.z)
startFish
user.startAnimation "position", user.container.chasePosMap, , _
  A_SPLINE+A_LOCAL, .001
user.startAnimation "direction", user.container.chaseDirMap, , _
  A_LINEAR+A_LOCAL, .001
world.trace "user chase animations started at time " & _
  world.getanimationsystemtime() & " ticks."
end sub
