Resource Desk FAQs

Q: What do I need to know to host a Virtual Worlds server for clients on the Internet?
A: Note that having any server directly available on the internet can expose you to potential security risks. To minimize risks you should consider taking some or all of the following measures:
  • Do not connect your external vworlds server to your internal network to minimize the exposure of other networked computers in the event of a security breach.
  • Limit the number of other services you run on the server machine to minimize impact/risk to your VWorlds server.
  • Ensure that your machine is up to date on all security patches and bulletins for your operating system and web server software.
  • If you are not hosting a public world you should restrict access. To restrict access to your worlds run the VWorlds server on a Windows 2000 (or NT4) server machine and enable NTLM authentication via the VWorlds server's Tools menu. Then, issue domain or local machine accounts to the users who will be accessing the machine. If there are only a subset of the users within the domain or local accounts who should have access to your VWorlds server you can further restrict access by creating a Local Group and adding just those specific accounts who should have access. Again, enable this Local Group feature via the server's Tools menu. [Note: Local Groups are created using the User Manager under NT4, the Computer Management administrative tool on Windows 2000, or the Active Directory Users and Groups tool for a Windows 2000 domain controller.]

Q: Where do I obtain support?
A: Your main source for support is the developer and user community. Please join the appropriate mailing list(s) and address your questions there. We will maintain our old Usenet news server while the new web site gets established, and then we'll migrate the posts over to our new web-based list archives. The archives will be an important resource, especially for new users. If you have specific questions for the vworlds.org team or cannot obtain the information you need, please contact info@vworlds.org.

Q: Will you allow commercial use?
A: No. The MSRSL and other licenses are focused on our research partners and do not allow commercial use of any portion of the SDK. When users request a commercial license, we will review requests on a case by case basis. If you would like to license the platform for commercial use, please email us at info@vworlds.org. Note that some third-party source code and/or libraries may not be available for commercial use depending upon agreements we have with each entity.
