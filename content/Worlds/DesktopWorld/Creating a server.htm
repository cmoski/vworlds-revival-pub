Creating a server for DesktopWorld

To run the Virtual Worlds Server, you will need the following components:

Windows 95, Windows 98, Windows NT 4, or Windows 2000.
Personal Web Server 4.0 or Internet Information Services 4.0 or better. (Download from http://www.microsoft.com/windows/ie/pws.)
Microsoft Data Access Components 2.0 or better. (Download from http://www.microsoft.com/data/download.htm.)

These instructions assume you are using WinNT or Win2K and IIS.

  1. Virtual Worlds Installation

    2. Install the Virtual Worlds core files and DesktopWorld application to the server.
    Right-click on "My Computer" and select "Properties", then select the "Network Identification" tab.
    Note the name of your server, then close the Properties dialog. This is the name used in steps 3.A, 3.B, and 8.A.

  2. Directory Creation

    3. We recommend using the default directories for the BBS, user pages, and website. If you choose to use different directories, make sure to edit the path names in steps 3 and 4.
    1. Create a directory for the BBS database, such as "C:\DesktopWorld\Database".
      2. Move the BBS database (BBS.mdb) from "C:\Program Files\Microsoft Virtual Worlds\Local Content\Worlds\DesktopWorld\External Content\Database" to this directory.
    2. Create a directory for the user pages database, such as "C:\DesktopWorld\Database".
      3. For convenience, we used the same directory for both databases, but they could be separately located.
      Move the user pages database (UserPages.mdb) from "C:\Program Files\Microsoft Virtual Worlds\Local Content\Worlds\DesktopWorld\External Content\Database" to this directory.
    3. Create a Website directory for external content files, such as "C:\DesktopWorld\Website".
      4. Move the files and directories from "C:\Program Files\Microsoft Virtual Worlds\Local Content\Worlds\DesktopWorld\External Content\Website" to this directory.

  3. Editing Paths

    1. Edit the file "vbs\pathnames.vbs" located in the Website directory (see step 2.C).
      1. Set strWebContentPath to the location of the external content files (see step 2.C).
      2. Set worldLocation to the name of the world you are going to create (see step 6.B).
      3. Set worldCount to the number of worlds you want to run on your server. This sample supports multiple worlds using the same base name, such as dtw1, dtw2, etc.
      4. Set userImagesPath to the directory where you want to store any uploaded data.
      5. Set strBBSServer to the server hosting the BBS (see step 1.A).
      6. Set strUserPagesServer to the server hosting your user pages database (see step 1.B).
      7. Set strUserPagesPath to the directory location of the ASP files for user pages (see step 2.C).
        8. (In our sample, these locations are all on the same computer. You may wish to locate them separately.)
    2. Edit the file "vbs\pathnames.vbs" located in the SDK directory (C:\Program Files\Microsoft Virtual Worlds\Local Content\SDK\wizards\world\Kits\DesktopWorld).
      1. Set strWebContentPath to the location of the external content files (see step 2.C).
      2. Set worldLocation to the name of the world you are going to create (see step 6.B).
      3. Set userImagesPath to the directory where you want to store any uploaded data.
      4. Set strBBSServer to the server hosting the BBS (see step 1.A).
      5. Set strUserPagesServer to the server hosting your user pages database (see step 1.B).
      6. Set strUserPagesPath to the directory location of the ASP files for user pages (see step 2.C).
        7. (In our sample, these locations are all on the same computer. You may wish to locate them separately.)

  4. Database Setup

    5. The instructions below assume the BBS and UserPages databases are on the same computer. If they are not, each server will need a copy of the INI file, have the appropriate DSN set up, and have MSADC enabled (see “Setting up the Web service”).
    1. Move "msdfmap_DesktopWorld.ini" from the website directory (see step 2.C) to the Windows directory (C:\WINNT).
    2. Go to Start | Settings | Control Panel | Administrative Tools | Data Sources.
      3. Select the "System DSN" tab. Click <Add...> to create a DSN.
         Driver: Microsoft Access Driver (*.mdb)
         Data Source Name: DesktopWorldBBS
         Database: BBS.mdb (in the BBS directory)
      Click <Add...> to create a second DSN.
         Driver: Microsoft Access Driver (*.mdb)
         Data Source Name: DesktopWorldPages
         Database: UserPages.mdb (in the user pages directory)

  5. Web Service Setup

    6. Go to Start | Settings | Control Panel | Administrative Tools | Internet Services Manager.
    Expand the tree and select "Default Web Site".
    1. Right-click on "MSADC" and click "Properties".
      2. Go to the "Directory Security" tab and click <Edit...> for IP restrictions.
      Change the default from "Denied" to "Granted" and click <OK>.
      Click <OK> to close the properties dialog.
    2. Right-click on a blank line and click "New" then "Virtual Directory".
      3. Click <Next>.
      Provide an alias. We use "Website" as our virtual directory name.
      Click <Next>, then click <Browse...> to find the directory containing the external content.
      Click <Next>, <Next>, and <Finish> to complete the setup.
    3. Right-click on a blank line and click "New" then "Virtual Directory".
      4. Click <Next>.
      Provide an alias. We use "Database" as our virtual directory name.
      Click <Next>, then click <Browse...> to find the directory containing the database file(s).
      Click <Next>, <Next>, and <Finish> to complete the setup.
    Close the IIS manager applet.

  6. World Setup

    7. Go to Start | Programs | Microsoft Virtual Worlds | Virtual Worlds Server.
    1. Go to Tools | User Authentication...
      2. Unselect "Require NTLM Authentication for Logins" then click <OK>.
    2. Go to File | New World...
      3. Enter the name of the world (see step 3.A), followed by the world number ("1", "2", etc.), then click <Next>.
      Provide a user name (do not set a password).
      Unselect “Automatically logon after wizard has finished.”, then click <Next>.
      Click <Next> (do not set a Content server).
      Select “DesktopWorld” as the type of world.
      Click <Finish> to build the world.
    3. Repeat step B to create multiple worlds.
      4. Remember to increment the world number ("2", "3", "4"...).

  7. Uploading Files

    8. WARNING: Enabling this DLL will allow users to upload files to your server. Currently, the script only permits .GIF and .JPG extensions, but does no other checking.
    1. Move FileUpload.dll from the website directory to C:\WinNT\system.
    2. Go to Start | Run and execute the following command: regsvr32 fileupload.dll.

  8. Starting the Client

    1. Go to the program directory (C:\Program Files\Microsoft Virtual Worlds\Local Content\Worlds\DesktopWorld\Client)
      2. Edit “html\launcher.htm” in the editor of your choice.
      View the HTML source, and find 'loginPath = "http://DesktopWorld/website/Login/"'.
      Change "DesktopWorld/website/" to the name of the server and directory created in steps 5.B.
      Save the file and close the editor.
    2. Open "launcher.htm" by double-clicking on it or right-clicking on it and selecting "Open".
      3. You can also create an Active Desktop item from “launcher.htm” with the following steps:
      1. Drag “launcher.htm” from Explorer onto the desktop using the Right mouse button. When you release the right mouse button, select “Create Active Desktop item(s) Here”.
      2. Click <Yes> to add the item to the desktop.
      3. Right-click on the desktop and select Active Desktop | Show Web Content if it is not selected.
      4. Click on the Active Desktop item to start the DesktopWorld client.
    3. Select the world to enter (1, 2, etc.), select or create a new nickname, then select a gender. Click <Go> or press <ENTER> to log into your world.
